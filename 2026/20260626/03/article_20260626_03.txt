Fine-tune、RLHF、RLAF 三级跳——从“背答案”到“人类打分”到“AI 打分”的模型定制进化

把基础模型想象成一个刚读完所有教科书的新人——什么都知道一点，但不知道怎么在具体公司干活。Fine-tune 是给他一本标准答案集，RLHF 是让老师傅反复打分，RLAF 是让一群 AI 质检员接管评估台。三级跳的核心问题只有一个：反馈信号从哪里来，能有多可靠，能有多大规模。

Fine-tune：给新人一本标准答案集

预训练结束后的基础模型，脑子里装着全网文本，但不知道怎么把知识组织成你想要的回答格式。Fine-tune 的做法很直接：收集一批“好的输入-输出对”，让模型反复看这些例子，调整内部权重，直到它遇到类似问题时倾向于输出类似风格和质量。

这个过程的本质是模仿。新人拿到了一本厚厚的 FAQ 手册，每条问题后面跟着标准答案。他认真背下来，再遇到手册里有过的提问，能对答如流。但手册没覆盖的场景，他立刻回到泛泛而谈的模式——他知道模仿标准答案的句式，但不知道为什么这个答案是好的。

技术上看，Fine-tune 调整的是模型几十亿个参数中的一部分，调整幅度通常不到 1%。但这微小的偏移足以改变输出分布。一个重要的工程选项是 LoRA——不碰全部参数，只在模型旁边插入一个几百万参数的“补丁”，只训练这个补丁。最终得到一个小文件，几十到几百 MB，搭配原始基础模型使用，效果接近全参数微调，成本却低一两个数量级。

普通人可行性很高。一张 24GB 显存的消费级 GPU，用 Ollama + Unsloth 或 LLaMA-Factory，几百条数据就能对 7B-14B 模型做 LoRA 微调。但这个方法的天花板也肉眼可见：反馈信号是静态的，数据准备好后不再变化，规模取决于你能准备多少问答对。学到的是“模仿模式”，不是“判断力”。

RLHF：让人类师傅反复打分

RLHF 改变了一件事：不再给标准答案，而是让模型反复尝试，每次尝试后人类师傅说“这次好”或“这次不行”。新人不再背手册，他开始自己写回答，写完交给师傅看，师傅打分，他根据分数调整下一次的写法。

具体流程分四步。第一步，模型对同一个问题生成多个候选回答。第二步，人类标注员把这些回答从好到坏排序。第三步，用排序数据训练一个奖励模型——这个奖励模型学会了模拟人类标注员的偏好，能对任意回答打出一个偏好分数。第四步，用强化学习算法根据奖励模型的评分调整原始模型权重，让模型倾向于生成高分回答。

这是 ChatGPT 比 GPT-3 好用得多的关键。GPT-3 做过 Fine-tune，但回答经常跑偏、啰嗦、不懂什么时候该闭嘴。RLHF 让模型学会了一种“分寸感”——不是记住哪个答案对，而是内化了一套关于有用性、安全性、礼貌程度的偏好。新人学会的是“判断力”而不是“记忆力”。

但代价极高。人类标注员一天只能标注几百条，而一轮 RLHF 训练可能需要几十万条排序数据。不同标注员的偏好还不一致——有人觉得详细好，有人觉得简洁好。数据成本动辄数百万美元，而且每轮训练都需要新的标注，反馈信号虽然动态，却无法规模化。普通人根本玩不起。

RLAF：AI 师傅接管评估台

RLAF 把人类师傅换成 AI 师傅。流程不变——模型生成多个候选回答，打分，强化学习更新权重——但打分的不再是人，而是一个或多个 AI agent。关键的进化在于：agent 能做人类做不到的事。

用一个供应链安全分析的场景串起来。假设你要训练一个安全分析模型。传统 Fine-tune 的做法是收集 10,000 条安全问答对，训练 3-5 轮，得到一个会说安全黑话的模型，但遇到新型攻击手法可能说不清楚。RLAF 的做法完全不同：先在基础 Fine-tune 之后进入强化学习阶段。准备 1,000 个安全问题，每个问题让模型生成 4 个候选回答。然后部署三个 agent——Agent A 查 NVD 和 OSV 数据库，验证 CVE 编号、受影响版本、修复状态是否准确；Agent B 评估分析覆盖面的完整性，检查是否遗漏了攻击向量或受影响组件；Agent C 直接运行回答中给出的修复命令，检查能不能执行、有没有副作用。三个 agent 各自打分，综合评分后更新模型权重，多轮重复直到收敛。

最终得到的模型不只是能说出术语，而是能给出准确、完整、可操作的安全分析。Agent C 能提供人类标注员根本无力提供的反馈——大部分标注员不会写代码，更不可能在沙箱里跑一遍修复命令。Agent A 能实时查询漏洞数据库，准确率远超人类记忆。一个 agent 一天可以评估几十万条回答，成本是人类的百分之一。

普通人可行性正在打开。用强模型作为评估 agent 就是简化版 RLAF——你可以让 Claude 或 GPT-4 给你的模型打分。开源框架 TRL、OpenRLHF 已支持 AI 反馈替代人类反馈。成本主要在 GPU 训练和 API 调用，比 RLHF 低很多，但比纯 Fine-tune 高。一张 24GB 显卡能做 LoRA 微调，但要跑 RLAF 的强化学习循环，可能需要多张卡或云算力。

反馈信号的三级进化与人类教育的镜像

三种方法的核心区别在于反馈信号的性质和规模。Fine-tune 的反馈是静态的标准答案——可靠但狭窄，规模几千到几万条，学到的是“模仿模式”。RLHF 的反馈是动态的人类偏好——更灵活但昂贵且不稳定，规模受限于人力，学到的是“人类价值观和偏好”。RLAF 的反馈是动态的 AI 评估——可规模化且多维，规模可达几十万条，学到的是“多维度质量标准”。

更深层看，这条三级跳背后有一个认知规律：当我们在教“什么是好的”时，核心瓶颈不是教学方法，而是反馈信号的可靠性和可规模化。Fine-tune 靠标准答案，可靠但覆盖面窄。RLHF 靠人类偏好，覆盖面广了，但信号噪声大、成本高。RLAF 靠 AI 评估，规模和维度都上去了，但引入了新问题——打分 agent 自己也会犯错、有偏见、被 reward hacking 欺骗。模型可能学会用花哨的格式骗过高分 agent，而不是真正提高质量。

这和人类教育的进化逻辑惊人一致。学徒制时代，师傅一对一给徒弟打分，反馈精准但无法规模化。标准化考试时代，标准答案评估一切，能覆盖所有人但扼杀创造力。多维度综合素质评价时代，多个老师从不同角度评分——但老师自己也有偏见，评估标准本身也需要被评估。AI 训练走的是同一条路，从背答案到师傅带徒弟到综合评估，连技术名词都不用换。

RLAF 不是终点，而是一个新的均衡点。它把成本从人类劳动转移到算力和 API 调用，把评估从单维偏好变成多维质检。但每个维度都需要一个能力足够强的 agent，而这些 agent 本身也需要被对齐。这是一个不断迭代的循环——你用今天的强模型训练明天的更强模型，而打分 agent 的偏见会像影子一样跟着这个循环走下去。


RLAF 把人类标注员踢出了训练循环，但把一个新的问题踢了回来：谁来给打分 agent 打分？如果一个 agent 检查代码可运行性，它自己的判断标准是谁训练的？如果多个 agent 综合评分，权重是谁定的？你会发现，RLAF 并没有消除“人类偏好”这个终极锚点，只是把它往后推了一层。推到哪一层为止？推到那个负责设定评估维度的工程师为止。所以别被“全自动 AI 训练”的说法迷惑——链条的起点，永远坐着一个人。
